# Database Design

## Overview

ActionPilot uses PostgreSQL as its primary database.

The database is designed around the lifecycle of a goal:

Goal Creation

↓

Planning

↓

Execution

↓

Progress Tracking

↓

Risk Evaluation

↓

Recovery

↓

Completion

The schema focuses on simplicity, maintainability, and rapid development for the hackathon MVP.

---

# Core Entities

The MVP consists of the following primary entities:

1. Users
2. Goals
3. Tasks
4. Progress Logs
5. Notifications
6. Agent Reports

---

# Database Design Summary

The ActionPilot database is centered around goals, tasks, progress tracking, notifications, and agent-generated insights.

The schema prioritizes simplicity while providing enough structure to support planning, monitoring, accountability, risk detection, and recovery workflows.
