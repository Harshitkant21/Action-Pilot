# Product Principles

> **Version:** 1.0  
> **Status:** Living Document  
> **Applies To:** Entire ActionPilot Platform

---

# Vision

ActionPilot is **not** a task manager.

It is an **AI Execution Companion**.

Traditional productivity tools help users organize work.

ActionPilot helps users **finish** work.

Every feature added to the platform must strengthen this vision.

---

# Mission

Help people consistently execute their goals by combining AI planning, continuous monitoring, intelligent coaching, proactive intervention, and adaptive recovery.

The platform should reduce the gap between **planning** and **execution**.

---

# Product Philosophy

The product should feel like working alongside a highly experienced project manager who:

- understands the user's goals
- notices problems early
- recommends better decisions
- adapts when plans fail
- celebrates progress
- never overwhelms the user

---

# Core Principles

## Principle 1 — AI Assists, Humans Decide

AI should recommend.

Users should decide.

The AI must never silently change user work without approval.

Examples

✔ Suggest recovery plans

✔ Suggest priorities

✔ Recommend deadlines

✘ Delete tasks automatically

✘ Rewrite plans without permission

---

## Principle 2 — Be Proactive, Not Reactive

ActionPilot should never wait for users to discover problems.

The system should proactively detect:

- deadline risks
- inactivity
- blockers
- declining confidence
- execution slowdown

Then surface these immediately.

---

## Principle 3 — Every Notification Must Be Actionable

Notifications should never exist only to inform.

Every notification should help users take action.

Examples

Risk Alert

→ Generate Recovery Plan

Standup Reminder

→ Open Check-in

Goal Complete

→ Review Summary

Recovery Ready

→ Compare Plans

---

## Principle 4 — Explain Every AI Decision

Every AI recommendation should answer:

Why?

Examples

Instead of

> Risk Score: 82%

Say

> Risk increased because progress slowed, confidence decreased, and remaining work exceeds available time.

AI should always be transparent.

---

## Principle 5 — User Control Comes First

The user owns their workflow.

Everything that affects their productivity should be configurable.

Examples

Standup schedule

Notification channels

Risk thresholds

Recovery behavior

Planning depth

AI coaching style

Timezone

Quiet hours

Dashboard widgets

The product adapts to the user.

The user should never adapt to the product.

---

## Principle 6 — Graceful Failure

AI will occasionally fail.

The product must never fail because AI failed.

Fallbacks should exist for:

Goal planning

Recovery

Risk analysis

Standups

Notifications

The user should always be able to continue working.

---

## Principle 7 — Make Intelligence Visible

AI should not operate invisibly.

Users should see:

AI Planning

Risk Analysis

Recovery Suggestions

Standup Coach

Execution Monitoring

Notifications

Background Monitoring

The product should continuously communicate its intelligence.

---

## Principle 8 — Preserve User Work

User-created work is the highest priority.

Recovery plans

Planning

Scheduling

Task updates

must never destroy completed work.

Completed tasks are immutable.

---

## Principle 9 — Measure Execution, Not Activity

ActionPilot should optimize for execution.

Metrics should focus on:

Goals Completed

Recovery Success

Execution Velocity

Planning Accuracy

Consistency

Risk Reduction

Avoid vanity metrics.

---

## Principle 10 — Continuous Improvement

The AI companion should improve over time.

Future versions should learn from:

execution history

risk patterns

completion behavior

preferred schedules

planning accuracy

while always respecting user privacy.

# Non-Goals

ActionPilot is NOT trying to become:

❌ Another Todo App

❌ Another Note-taking Tool

❌ Another Calendar

❌ Another Chatbot

❌ Another Documentation Tool

Instead it complements these products by becoming the execution layer above them.

---

# Decision Framework

Before implementing any new feature, ask:

Does it improve execution?

Does it reduce cognitive load?

Does it help users finish work?

Does it make AI more visible?

Does it keep users in control?

Can users configure it?

Does it preserve existing workflows?

If most answers are **No**, the feature should not be implemented.
